The SolScanner Protocol

A decentralized framework for real-time ledger forensics and heuristic risk analysis on the Solana Network.

AUTHORS: 0xArch, Vex DATE: August 2024

1. Abstract

The proliferation of high-throughput blockchains has created a data asymmetry between malicious actors and retail participants. SolScanner addresses this by introducing Sentinel Nodes, a layer of validator-adjacent observers that index state changes in milliseconds. This paper outlines the architecture of our Recursive Metadata Analysis (RMA) engine.

2. The Latency Problem

Standard RPC nodes prioritize transaction broadcasting over query speed. By the time a standard getAccountInfo call returns data about a liquidity removal event, the block has already been finalized. SolScanner utilizes the Geyser plugin interface to stream account updates directly from the validator runtime to our analysis engine via gRPC.

// Latency Comparison Formula
T_detect = T_block + T_indexing + T_query
Standard RPC: 400ms + 2000ms + 150ms = ~2.5s
SolScanner: 400ms + 10ms + 20ms = ~430ms

3. Risk Heuristics

We assign a deterministic risk score $R$ based on a weighted vector of mutable properties. The core equation is defined as:

R = 100 - (w_m * M + w_f * F + w_l * L)

Where $M$ is Mint Authority presence, $F$ is Freeze Authority, and $L$ is Liquidity Lock status. The weights $w$ are adjusted dynamically via DAO governance based on evolving scam patterns.

4. Tokenomics ($SCAN)

The $SCAN token serves as a work-utility token. Enterprise clients must stake $SCAN to access higher API rate limits. Furthermore, 30% of protocol revenue is used to buy back and burn $SCAN, creating a deflationary pressure correlated with network usage.

5. Conclusion

By commoditizing forensic data, SolScanner aims to restore transparency to the Solana ecosystem, making it safer for institutional capital deployment.
